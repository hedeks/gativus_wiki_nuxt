# Аудит лишних и legacy файлов

Документ фиксирует кандидатов на удаление или перенос на основе фактического
состояния репозитория и направления, заданного в `manifests/wiki_feature_plan.md`.

## Критично (безопасность)

- `little_secret.md`
  - Содержит секрет в открытом виде.
  - Рекомендация: удалить из репозитория, заменить секрет, хранить в `.env`.

## Дубли / потенциальный конфликт маршрутов

- `server/routes/sitemap.xml.ts`
- `server/routes/sitemap.xml.get.ts`

Обе реализации относятся к sitemap. Одна основана на Nuxt Content, другая на БД.
Рекомендация: оставить одну каноничную реализацию под текущую data-модель.

## Legacy-контур (кандидаты на удаление после миграции ссылок)

### Страницы

- `pages/courses/index.vue`
- `pages/courses/[...slug].vue`

### API

- `server/api/lection/index.get.ts`
- `server/api/lection/[...].get.ts`
- `server/api/quiz/index.get.ts`
- `server/api/quiz/[...].get.ts`

### Компоненты, где еще встречаются ссылки на `/courses`

- `components/theFooter.vue`
- `components/theCourse.vue`
- `components/theProfileIcon.vue`

## Что уже видно по git-изменениям

В рабочем дереве уже удалены части старого контента:

- `content/courses/book-gtom.md`
- несколько файлов из `data/quizez/*.json`
- `content/about.md`

Это подтверждает, что legacy-файловый контур уже демонтируется.

## Рекомендуемый безопасный порядок очистки

1. Удалить/ротацировать секреты.
2. Перевести все UI-ссылки с `/courses` на актуальные маршруты.
3. Проверить, что legacy API больше не вызываются страницами/компонентами.
4. Удалить `pages/courses/*` и `server/api/lection/*`, `server/api/quiz/*`.
5. Оставить одну реализацию sitemap и проверить выдачу `/sitemap.xml`.
6. Провести smoke-тест публичных и админских маршрутов.

## Примечание

`manifests/*.md` и `plans/*.md` не являются runtime-файлами приложения, но
полезны как проектная документация. Их не стоит считать "мусором" автоматически.
