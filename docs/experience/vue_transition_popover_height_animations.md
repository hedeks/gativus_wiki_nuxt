# Плавная анимация высоты попапов с Vue Transition

## Проблема
При разработке UI-попапа (`theTermPopover` и `KnowledgeGraphVisualizer`) для терминов и энциклопедических карточек была выявлена проблема: при асинхронной загрузке данных (скелетон -> реальный контент) высота попапа менялась скачкообразно. Использование встроенных в Vue 3 хуков `<Transition>` вызывало мерцания и подпрыгивания контента при переключении `v-if`. Дополнительная проблема заключалась в том, что когда попап позиционировался выше курсора (через `top`), при изменении высоты его нижний край отрывался от курсора, и попап улетал вверх по экрану.

## Решение
Был реализован архитектурный подход к плавному кроссфейду и анимированию высоты, который позволяет избежать любых "дерганий" интерфейса:

1. **Обертка контента в Transition с абсолютным позиционированием**:
   - Контент был обернут в `<Transition name="fade">`.
   - В CSS был добавлен класс `.fade-leave-active { position: absolute; top: 0; left: 0; width: 100%; pointer-events: none; }`. 
   - Это гарантирует, что уходящий элемент (например, скелетон) мгновенно изымается из нормального потока документа, позволяя контейнеру "обхватить" высоту *нового* элемента (контента), не складывая их высоты вместе во время фазы transition.

2. **Синхронный замер высоты (без магических чисел и хардкода)**:
   - В хуке `onEnter(el)` использован паттерн "мгновенного замера" (DOM reflow hack):
     \`\`\`typescript
     const origHeightStyle = popup.style.height;
     popup.style.height = 'auto'; // Снимаем жесткую высоту
     const newHeight = popup.offsetHeight; // Считываем натуральную высоту с учетом паддингов
     popup.style.height = origHeightStyle; // Возвращаем обратно до рендера кадра
     void popup.offsetHeight; // Форсируем Reflow
     \`\`\`
   - Этот подход навсегда избавил от необходимости хардкодить `34px` (для паддингов/бордеров), так как `offsetHeight` возвращает точную физическую высоту компонента.

3. **Синхронизация высоты и позиции (top)**:
   - Чтобы попап, открытый *сверху* от курсора, не прыгал при изменении высоты, к нему применяется CSS-переход: `transition: height 0.2s cubic-bezier(...), top 0.2s cubic-bezier(...)`.
   - Важно: обе анимации (и `height`, и `top`) используют **одинаковую кривую Безье и длительность**. Благодаря этому, пока высота уменьшается/увеличивается, `top` синхронно сдвигается, сохраняя положение нижнего края ровно на курсоре.

## Извлеченные уроки (Ловушки)

1. **Снятие стилей перед рендером (removeProperty)**:
   - Была выявлена ловушка: если функция позиционирования вручную меняет `el.style.top`, а в конце удаляет его через `el.style.removeProperty('top')` (оставляя стили только в реактивном Vue-объекте `popoverStyle`), это ломает CSS Transition.
   - Браузер запускает анимацию до того, как Vue применит реактивный объект в следующем тике, из-за чего стартовой точкой анимации становится `top: 0` (или `auto`). 
   - *Урок*: Никогда не используйте `removeProperty` для инлайн-координат перед стартом CSS-анимации, если эти координаты дублируются в Vue-реактивности. Доверяйте Vue перезапись инлайн-стилей.

2. **Media и Aspect-Ratio**:
   - При загрузке асинхронных изображений без заранее известной высоты (`height: auto`), по завершении Transition произошел бы еще один скачок, когда картинка скачается из сети.
   - *Урок*: Всегда резервируйте место под медиафайлы в таких попапах либо через фиксированную высоту для медиа-контейнера (`height: 140px; object-fit: cover;`), либо через `aspect-ratio`, чтобы предотвратить изменение высоты DOM-дерева после завершения хуков Vue.
