# Редизайн экскурсий в Knowledge Graph (Часть 2)

## Проблема
Пользователь обратился с задачей интегрировать логику воспроизведения экскурсий (ранее назывались Storytelling/Маршруты) непосредственно в карточки узлов на графе, удалив отдельную устаревшую панель. Были выявлены следующие проблемы:
1. Старые компоненты `FeaturedStoriesPanel` и `StorytellingPanel` засоряли код и не вписывались в обновленный UI графа.
2. Маршруты не поддерживали вывод авторских заметок прямо в контексте текущего шага.
3. В админке возникали ошибки авторизации при сохранении и проблемы с версткой.

## Решение
1. **Удаление легаси**: Компоненты `FeaturedStoriesPanel` и `StorytellingPanel` полностью удалены, а их логика воспроизведения перенесена внутрь `KnowledgeGraphVisualizer.vue`.
2. **Интерфейс запуска**: Добавлено модальное окно `ExcursionsModal.vue` для удобного выбора экскурсии и отдельная кнопка-триггер в основном тулбаре графа (со звездочками).
3. **Плеер в попапе**: Логика шагов экскурсии интегрирована в `.graph-popup-node`. Если экскурсия активна, в самом начале попапа выводится плашка `Заметка автора` (берется из словаря `custom_messages` по текущему индексу шага). 
4. **Управление**: В футер карточки узла добавлены кнопки навигации (Назад, Далее, Завершить). Управление полностью поддерживает клавиатуру (Escape для выхода, Пробел/Enter/Стрелки для продвижения).
5. **Локализация**: В `uiDict` добавлены необходимые фразы (`guidedTours`, `excursionNote`, `exitExcursion` и т.д.) на трех языках (en, ru, zh).

## Извлеченные уроки и ловушки
- **Дублирование функций Vue**: При замене больших блоков кода через скрипты (или автозамены) легко случайно продублировать методы (например, `nextStoryStep`), что приведет к ошибкам компиляции Vite `Identifier 'nextStoryStep' has already been declared`. Всегда нужно проверять контекст до и после вставки.
- **Глобальные слушатели `keydown`**: При добавлении кастомной логики навигации (например, для экскурсии) важно перехватывать нажатия (например, `ArrowRight`, `Space`) только если активна экскурсия, а иначе пропускать их до дефолтных обработчиков (перемещение фокуса по узлам графа), не забывая при этом делать `e.preventDefault()`.
- **Авторизация в Nuxt (FetchError 401)**: При вызове серверных эндпоинтов админки с клиента (через `$fetch`) на клиенте не добавляются куки или токены автоматически, если используется схема Bearer JWT. В `$fetch` необходимо вручную передавать `headers: store.getAuthHeader()`, иначе сервер вернет ошибку 401 Unauthorized. (Эта доработка в админке оставлена на Part 3).
