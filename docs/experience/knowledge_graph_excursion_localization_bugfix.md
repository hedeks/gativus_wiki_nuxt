# Локализация экскурсий графа и фикс попапов рёбер

## Проблема
1. **Локализация**: Режим экскурсий (Storytelling mode) и всплывающие окна (попапы) графа знаний имели захардкоженные тексты (например, «Author's notes»). Блок `Notes` использовал грязные хаки вида `.replace('раз в статье', '')` для подгонки падежей, что было непереносимо на другие языки (например, китайский).
2. **Навигация в экскурсиях**: В режиме экскурсии внутри попапов рендерились стандартные контекстные стрелки (Вверх/Вниз/Влево/Вправо) для графа, которые ломали логику экскурсии.
3. **Рассинхронизация экскурсии**: Если пользователь во время экскурсии нажимал на соседний узел, который тоже есть в экскурсии, текущий шаг (`storyCurrentIndex`) не обновлялся, из-за чего навигация запутывалась.
4. **Неоткрывающиеся попапы рёбер**: Попапы для связей на основном графе перестали открываться. Клик по ребру происходил, но попап моментально закрывался.

## Решение
1. **Очистка шаблонов и расширение `uiDict`**:
   - Словарь `uiDict` внутри `KnowledgeGraphVisualizer.vue` расширен ключами для всех трех языков (`en`, `ru`, `zh`) для всех стрелочек, шагов экскурсий и сложных фраз.
   - Из шаблона полностью удалены вызовы `.replace()`: вместо этого использована структура `<template v-if="...mentionCount">`, которая собирает фразу из маленьких, независимых от падежей ключей (`t.notesMentioned`, `t.notesTimesInArticle`).
2. **Замена стрелочек навигации**:
   - В шаблоне добавлена логика `v-if`: если `storyState` активен, скрываются стрелки "Вверх" и "Вниз". Стрелки "Влево" и "Вправо" переназначаются на вызовы `prevStoryStep()` и `nextStoryStep()`.
3. **Синхронизация шага**:
   - В обработчике `on('click')` для узлов D3 (как для `force simulation` графа, так и для DAG ego-графа) добавлено вычисление индекса узла в `storyPath`. Если он там есть — `storyCurrentIndex` принудительно синхронизируется.
4. **Исправление глобального обработчика**:
   - Глобальный обработчик `handleOutsideClick` (навешанный на `window`) проверяет `event.target`. Добавлено исключение `if (target.closest('.link-hit')) return`, чтобы клики по невидимым утолщенным хит-путям связей не воспринимались как "клик вне графа".

## Извлеченные уроки (Ловушки)
1. **Глобальные обработчики vs D3 SVG**: При реализации `click-outside` директив или функций на `window` в проектах с D3 нужно быть предельно осторожными с SVG-элементами. Разные представления графа могут использовать разные теги (например, `<line>` в одном месте и `<path>` в другом). Все кликабельные элементы SVG должны иметь унифицированный класс-маркер (например, `.link-hit`), чтобы глобальный слушатель мог корректно их игнорировать через `target.closest()`.
2. **Опасность `.replace()` в локализации**: Использование модификации переведенных строк "на лету" с помощью `replace()` — крайне плохая практика. В других языках слова могут иметь другую длину, структуру или вообще не иметь пробелов (как в китайском), что приведет к тому, что `replace()` просто не сработает и сломает UI. Всегда лучше дробить сложную фразу на несколько переменных или использовать полноценные функции локализации с плюрализацией.
