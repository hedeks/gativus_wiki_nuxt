# Экспорт и Импорт в формате ZIP с предпросмотром в памяти

## Проблема
Пользователь обратился с необходимостью переработать систему бэкапов и синхронизации. Ранее экспорт и импорт производились только в формате чистого `JSON`. Это не позволяло переносить вместе со статьями привязанные медиафайлы (изображения, презентации, видео). При импорте на другой инстанс все ссылки на файлы оказывались битыми, так как сами файлы физически не переносились.

## Решение
1. **Экспорт в ZIP**: Логика бэкапа была переработана с использованием библиотеки `adm-zip` на стороне сервера (Node.js/Nitro). Теперь при экспорте генерируется ZIP-архив, куда помещается `data.json` с графом знаний и все физические файлы из директории `server/storage/uploads/`, которые фактически упоминаются в базе данных.
2. **Импорт (ZIP/JSON)**: Интерфейсы импорта были адаптированы под прием как старых `.json` файлов, так и новых `.zip` архивов.
3. **Безопасный предпросмотр без сохранения на диск**: Для визуализации графа перед импортом был создан эндпоинт `POST /api/admin/sync/preview`. Вместо того, чтобы парсить архив на клиенте (что потребовало бы добавления тяжеловесной зависимости вроде `jszip` во фронтенд-бандл), мы отправляем файл на сервер через `FormData`.
   - В `h3` мы читаем форму через `readMultipartFormData`.
   - Файл остаётся в оперативной памяти (в виде `Buffer`), на диск не сохраняется.
   - `adm-zip` читает архив из буфера, находит `data.json` и список остальных медиафайлов.
   - Сервер возвращает клиенту JSON-дамп и массив с названиями ассетов, после чего временные данные в памяти уничтожаются.
4. **Отображение ассетов**: Под канвасом графа теперь рендерится список прикрепленных медиафайлов, чтобы администратор видел, какие файлы приехали в бэкапе.
5. **Фикс стилей LanguageSwitcher (Fast-track)**: Убрано сужение и перестроение элемента на мобильных экранах, теперь переключатель языков сохраняет десктопный вид на любой ширине.

## Извлеченные уроки (Ловушки)
1. **Чтение архивов из буфера в H3 (Nitro)**: Чтобы не захламлять файловую систему временными файлами для предпросмотра, можно передать `fileField.data` (Buffer) напрямую в `new AdmZip(buffer)`. Это работает очень быстро и позволяет извлекать нужные файлы на лету (например, `entry.getData().toString('utf-8')`).
2. **Конфликт высот flex-контейнеров и абсолютного позиционирования**: Компонент `KnowledgeGraphVisualizer` имеет `min-height: 400px;`. Если положить его в контейнер с ограничением по высоте меньше 400px (например, `h-[350px]`) без указания `overflow-hidden`, он выйдет за границы родителя и визуально наслоится на следующие за ним элементы (например, на список ассетов). Решение: либо давать контейнеру высоту равную `min-height` дочернего элемента, либо жестко скрывать переполнение (`overflow-hidden`), а следующему блоку задавать новый контекст наложения (`relative z-10`).
3. **Отсутствие нужды в adm-zip на фронтенде**: Node-модули вроде `adm-zip` завязаны на `fs` и работают только на бэкенде. Попытки заставить их работать во Vue-компонентах приведут к ошибкам сборки Vite. Все операции с ZIP лучше делегировать серверу или использовать специальные клиентские библиотеки (например, `fflate` или `jszip`), если сервер не подходит. В нашем случае in-memory парсинг на сервере оказался элегантнее.
