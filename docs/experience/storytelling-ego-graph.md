# Опыт: Внедрение Storytelling Mode и навигации (Context Mode) в Графе Знаний

## Проблема
Пользователю требовался режим последовательного и управляемого погружения (Storytelling Mode) для узлов графа, а также возможность навигации по иерархии (вверх-вниз по контексту: Категория -> Книга -> Статья -> Термин) прямо из попапа графа. Изначально подход заключался в применении CSS-класса `.dimmed` ко всем узлам графа, кроме выделенных, что вызывало сильнейшие просадки производительности из-за массивных манипуляций с DOM и большого количества узлов D3.

## Решение
Был реализован **Ego Graph** подход вместо манипуляций стилями основного графа:
1. **Машина состояний (FSM)**: Введен `storyState` (`IDLE`, `STORY_START`, `PLAYING_NODE`, `CONTEXT_MODE` и т.д.), который детерминированно управляет состоянием графа и навигации.
2. **Ego Graph**: При переходе в режим истории (Storytelling/Context) основной макет графа перестраивается. Функция `renderDagEgoGraph` генерирует облегчённый DAG-ориентированный саб-граф, который содержит только узлы пути истории. Это полностью убрало лаги, так как не нужно затенять весь массив узлов через CSS-классы.
3. **Иерархическая навигация**: В попап были добавлены кнопки "Вверх" и "Вниз" для путешествия по иерархии связей (`contextUpNode`, `contextDownNode`).

## Извлеченные уроки и ловушки
- **CSS-манипуляции в D3**: Затенение сотен/тысяч узлов через `selectAll().classed('.dimmed')` — огромный антипаттерн производительности. Использование отдельной функции рендеринга только необходимых узлов (Ego Graph) — единственное рабочее решение для больших графов.
- **Позиционирование попапа (Схлопывание/Анимации)**: Когда мы привязываем попап через `top` и `left`, а затем анимируем изменение его высоты при загрузке новых данных (например, скелетон сменяется реальным текстом), попап начинает "улетать" вверх, если он находится над узлом. Динамическое переключение CSS-свойств (`bottom` вместо `top`, когда попап должен расти/сужаться вверх от нижней точки) не сработало стабильно из-за конфликтов с рефлоу. Эту проблему еще предстоит решить (запланировано перенятие проверенного решения из `articles/slug`).
- **Сброс Git-состояния (Важно)**: Ошибки инструментов автоматической замены кода не следует чинить через `git checkout`, если предварительно не были зафиксированы в коммиты все успешные изменения сессии. Спасением может служить только локальная история IDE. Изменения должны всегда коммититься инкрементально.

2. Добавить кнопки "Влево/Вправо" для перемещения по соседним узлам (siblings) одного уровня иерархии.
3. Окончательно решить проблему стабильного позиционирования и анимации попапа при смене высоты.
