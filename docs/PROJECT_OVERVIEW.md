# Gativus Wiki — Обзор проекта

## 1) Что это за проект

`gativus-wiki-nuxt` — Nuxt 3-приложение с публичной вики-частью и админкой.
Проект находится в активной миграции от старой схемы контента на файловой
системе (`content/courses`, `data/quizez`) к новой схеме на базе SQLite и
серверных API.

Целевая архитектура и функционал описаны в `manifests/wiki_feature_plan.md`.

## 2) Технологический стек

- `Nuxt 3` + `Vue 3`
- `@nuxt/ui` + `TailwindCSS`
- `Pinia` для клиентского состояния
- `Nitro server` (`server/api/*`)
- `better-sqlite3` + `useDatabase()` для локальной БД
- `@xmldom/xmldom` и `adm-zip` для ODT-импорта
- `d3` для графа знаний

## 3) Структура приложения

### Публичная часть

- `pages/index.vue` — главная
- `pages/about.vue` — страница о проекте
- `pages/books/index.vue`, `pages/books/[slug].vue` — книги и оглавление
- `pages/articles/index.vue`, `pages/articles/[slug].vue` — статьи
- `pages/glossary/index.vue`, `pages/glossary/[slug].vue` — глоссарий
- `pages/knowledge-graph.vue` — визуализация графа знаний

### Админка

- `layouts/admin.vue` — общий layout админ-панели
- `pages/admin/*` — управление книгами, статьями, глоссарием, категориями,
  импортом ODT и синхронизацией

### Сервер

- `server/api/auth/*` — логин/регистрация/хеширование
- `server/api/articles/*` — CRUD статей, ревизии, откат
- `server/api/books/*` — CRUD книг
- `server/api/terms/*` — CRUD терминов, поиск
- `server/api/categories/*` — дерево категорий и reorder
- `server/api/import/*` — импорт/превью ODT
- `server/api/search.get.ts` — поиск
- `server/api/knowledge-graph.get.ts` — данные графа

## 4) Наблюдаемое состояние миграции

В кодовой базе одновременно присутствуют две модели:

1. Новая wiki-модель (DB-first): активные страницы `/books`, `/articles`,
   `/glossary`, `/knowledge-graph`, админские CRUD/API, ODT импорт.
2. Legacy-модель (file-first): страницы `/courses`, API `server/api/lection/*`
   и `server/api/quiz/*`, чтение из файловой системы.

Это означает, что часть интерфейса и API уже работает по новой модели, а часть
по старой, что создает точки рассинхронизации и потенциальные 404/500.

## 5) Что важно выровнять в ближайшее время

- Полностью убрать ссылки на `/courses` и перейти на `/books`/`/articles`.
- Либо сохранить legacy-файлы как временный режим, либо удалить их после
  завершения миграции ссылок и данных.
- Убедиться, что sitemap реализован одной каноничной версией (без дублей route).
- Вынести любые секреты из репозитория в переменные окружения.

## 6) Итог

Проект уже содержит основную инфраструктуру вики (БД, API, админка, глоссарий,
граф знаний, импорт ODT), но не завершил уборку legacy-слоя. Текущий приоритет
— консолидация маршрутов и удаление остаточного файлового контура.
