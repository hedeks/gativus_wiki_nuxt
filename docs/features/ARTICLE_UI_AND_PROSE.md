# Интерфейс статьи и оформление HTML (`article-prose`)

Рендер контента статей и терминов с расширенным HTML из ODT.

---

## 1. Где подключается стиль

Глобальный файл: **`assets/css/article-prose.css`** (импорт из `app.vue`).

Класс-контейнер: **`.article-prose`** на элементе с **`v-html`** (основной текст статьи или термина).

Типичная разметка карточки (упрощённо):

- Внешняя карточка: `overflow-x-hidden`, чтобы **не** появлялась горизонтальная прокрутка всего блока из‑за широких таблиц.
- Внутри — `.article-prose` с **`overflow-x: visible`** — вертикальный ритм текста не привязан к общему горизонтальному скроллу.

---

## 2. Таблицы и горизонтальный скролл только у таблицы

Проблема: таблица с `width: max-content` расширяла весь блок текста и включала скролл на уровне всего prose.

Решение:

1. Утилита **`utils/wrapArticleTables.ts`** — функция **`wrapArticleTables(html)`** оборачивает каждый корневой `<table>...</table>` (с учётом **вложенных** таблиц) в:

   ```html
   <div class="article-table-scroll">…</div>
   ```

2. Вызов при подготовке HTML:
   - `pages/articles/[slug].vue` — после подсветки поиска в вычисляемом теле статьи.
   - `pages/glossary/[slug].vue` — для `article_html` / `definition`.

3. В CSS:
   - **`.article-prose .article-table-scroll`** — `overflow-x: auto`, `max-width: 100%`, отступы как у прежней таблицы (`margin: 2em auto`).
   - **`.article-prose .article-table-scroll table`** — `width: max-content`, `min-width: 100%`, шрифт на 2px меньше базового текста статьи (`calc(1em - 2px)`).
   - Ячейки: `overflow-wrap: anywhere`, `word-break: break-word`, **`min-width: 2.5em`**, чтобы колонки не схлопывались до пары пикселей.

Шапка таблицы (тёмно-синий фон), зебра строк, случай без `<thead>` (первая строка как шапка) — описаны комментариями в **`article-prose.css`**.

Превью в админке (`import`, `edit`) может **не** вызывать `wrapArticleTables` — там ширина контролируется отдельно; при необходимости функцию можно переиспользовать.

---

## 3. Оглавление (TOC)

Компонент: **`components/theToc.vue`**.

- На **`lg` и выше**: липкий блок с **прозрачным** фоном (`max-lg:` задаёт «стекло» и рамку только для плавающей мобильной панели).
- На мобильных: сворачиваемая панель справа сверху под шапкой.

Корректная работа якорей с разметкой из ODT (маркеры заголовков, порядок атрибутов в HTML) синхронизирована со страницами **`[slug].vue`** / **`glossary/[slug].vue`** и **`components/theTocItem.vue`** (scroll vs toggle для узлов без детей).

---

## 4. Прочие элементы в `article-prose`

- Заголовки `h1–h6`: цветовая шкала, `scroll-margin-top` под фиксированную шапку и мобильное TOC.
- Абзацы: выключка (`text-align: justify`), переносы агрессивные для длинных строк.
- Ссылки **`a.wiki-term`**: отдельный зелёноватый акцент (согласован с типом «term» на графе).
- Изображения: лайтбокс по клику (обработчики на странице статьи).
- Блоки ODT: `.odt-textbox`, `.odt-note`, маркеры `.odt-heading-marker`, `.odt-li-marker` и inline глав — см. конец **`article-prose.css`**.
