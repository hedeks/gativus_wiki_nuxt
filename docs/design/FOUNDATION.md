# Gativus Design Foundation

Этот документ фиксирует foundation, на котором построен текущий UI-рефактор (knowledge graph + admin), чтобы следующие изменения оставались визуально едиными.

## 1. Дизайн-принцип

**Premium minimalism**: спокойная глубина, четкая типографика, один акцентный цвет, чистая иерархия.

Опорные референсы в проекте:
- `pages/about.vue` — карточная структура (`section-card`, `card-header`, `card-body`)
- `pages/articles/[slug].vue` — чистая контентная поверхность и аккуратная подача деталей

## 2. Токены и глобальная база

Основные токены определены в `app.vue`:
- Поверхности: `--gv-surface`, `--gv-surface-card`, `--gv-surface-header`
- Текст: `--gv-text-primary`, `--gv-text-secondary`
- Границы/глубина: `--gv-border-principal`, `--gv-shadow-sm|md|lg`
- Акцент: `--gv-primary`, `--gv-primary-hover`
- Радиусы: `--gv-radius-container`, `--gv-radius-control`

Глобальные utility-классы:
- `gv-page` — основной контейнер страницы
- `gv-surface-card` / `gv-card-header` — базовая карточка
- `gv-hero-gradient` — брендовый акцент в hero-заголовках
- `gv-filter-pill` — единые фильтр-пиллы
- `gv-focusable` / `gv-focus` — видимый focus-state

## 3. Admin Foundation

Введена единая admin-база:

- `gv-admin-page` — унифицированный контейнер страниц админки
- `gv-admin-surface` — базовая поверхность для admin-блоков
- `gv-admin-head` — шапка секции
- `gv-admin-eyebrow`, `gv-admin-title`, `gv-admin-subtitle` — типографический набор шапок

Общий shell админки:
- `layouts/admin.vue`:
  - общий фон и поверхности переведены на `--gv-*` токены
  - единый внутренний контейнер `admin-content-inner gv-admin-page`
  - адаптирован topbar/sidebar для узких экранов

## 4. Смысловой цвет и онтология

Семантический цвет сохраняется как системный язык:
- Book: `#0ea5e9`
- Article: `#6366f1`
- Term: `#10b981`
- Category: `#ef4444`

Для линков в графе:
- `virtual` связь — dashed
- `structural` связь — solid

## 5. Mobile-first правила (обязательные)

На `<= 768px`:
- Шапки страниц становятся колонкой (title + actions)
- Action-блоки переходят в wrap/full-width
- Таблицы только через горизонтальный scroll-container
- Редакторы и длинные формы должны убирать жесткие `100vh`-режимы
- Tap-target кнопок не меньше ~40px

На `<= 480px`:
- уменьшать внутренние отступы карточек и topbar
- скрывать вторичный текст в тулбарах, оставляя иконки
- важные CTA делать full-width

На `<= 360px`:
- уплотнять admin-head и content-shell отступы
- дополнительно снижать размер title/subtitle без потери контраста
- action-группы переводить в one-control-per-row

## 6. Паттерны, которыми пользоваться дальше

### 6.1 Card pattern
- Внешний контейнер: `gv-surface-card`
- Верх: `gv-card-header`
- Контент: локальный body-блок с ровным vertical rhythm

### 6.2 Table pattern
- Обертка: поверхность + `overflow-x: auto`
- Таблица: компактный uppercase-head + мягкие row-dividers
- На мобиле не ломать колонки; только горизонтальный скролл

### 6.3 Form/Edit pattern
- Блоки формы как секции-карточки
- Явные заголовки секций
- Sticky/заметные primary actions внизу формы на мобиле (где уместно)

## 7. Правила для новых UI-изменений

1. Сначала использовать существующие `--gv-*` токены, потом добавлять новые.
2. Не вводить новый акцентный цвет без продуктового смысла.
3. Не дублировать локальные `.card`-стили, если можно переиспользовать foundation-классы.
4. Любой новый admin-экран сразу проверять на 360-390px ширине.
5. Если элемент интерактивный — обязателен `:focus-visible` state.

## 8. Быстрый чеклист перед PR

- Цвета и фон соответствуют `--gv-*`
- Card/table/form имеют единый ритм
- Нет “ломающихся” таблиц на телефоне
- Действия доступны большим пальцем
- Темная тема не теряет контраст
