# План по устранению дизайн-противоречий (Унификация UI/UX)

**Внимание исполнителю (LLM):** Ваша задача — устранить разногласия стилей (разные кнопки, цвета, закругления) по всему сайту и особенно в админ-панели. Текущий дизайн-код стремится к стилю "Premium/Minimalist". Вы должны строго следовать описанным ниже правилам и шагам. Не придумывайте новые стили, используйте только утвержденные.

## 🎨 Единая система дизайна (Core Design System)

Для обеспечения единообразия всех элементов используйте следующие константы:

1. **Радиусы закругления (Border Radius):**
   - Карточки и модальные окна: `15px` (`border-radius: 15px;` или `rounded-2xl` / `rounded-xl`). Избегайте острых углов.
   - Кнопки: `10px` или `12px` (`rounded-xl` в Tailwind).
   - Бейджи и теги: `8px` (`rounded-lg`).

2. **Кнопки (Buttons):**
   - **Главные действия (Primary):** Кнопка должна быть синей (`sky`). Использование: `<UButton color="primary" variant="solid" class="rounded-xl px-8" size="lg">`.
   - **Второстепенные действия (Secondary):** Серые кнопки. Использование: `<UButton color="gray" variant="soft" class="rounded-xl" size="lg">`.
   - **Отмена/Удаление:** Деструктивные действия `<UButton color="red" variant="soft">` или `variant="ghost"`.
   - *Важно:* Всегда используйте `color="primary"` для главных кнопок, так как он привязан к системному синему цвету.

3. **Цветовая палитра:**
   - Основной акцентный цвет: `sky` (светло-синий/голубой) для иконок, ховеров текста, активных табов.
   - Фон карточек: белый (`#fff`) для светлой темы, темно-серый/почти черный (`#18181b` или `zinc-900`) для темной.
   - Границы (Borders): светло-серые (`#c8c8c8` или `#e5e7eb`) для светлой темы, темные (`#2a2a2e` или `#27272a`) для темной.

4. **Типографика:**
   - Заголовки страниц (H1): Крупные (24-26px), жирные (800), UPPERCASE (все заглавные), `letter-spacing: 2px;`.
   - Подзаголовки блоков: 11-13px, жирные (700), UPPERCASE, `letter-spacing: 0.1em;`, цвет текста — серый (`#6b7280`).

## 🛠 Пошаговый план выполнения (Roadmap)

### Шаг 1: Аудит и исправление кнопок в Админ-панели
Пройдите по следующим файлам и замените все `color="primary"` или кастомные стили кнопок на стандартизированные:
1. `pages/admin/index.vue`
2. `pages/admin/articles/index.vue` и `pages/admin/articles/create.vue`
3. `pages/admin/glossary/index.vue` (убедитесь, что кнопки создания и редактирования выглядят одинаково).
4. `pages/admin/books/index.vue` и страницы управления оглавлением.
5. `pages/admin/categories/index.vue`.
**Чек-лист:** Все главные кнопки сохранения/создания должны быть `color="primary"`, `size="lg"`, `class="rounded-xl"`.

### Шаг 2: Унификация Карточек (Cards)
Многие страницы имеют разный стиль обрамления списков или форм.
В файлах форм (например, `pages/admin/articles/[id]/edit.vue` и `create.vue`):
1. Найдите обертки контейнеров (`.card`, `.form-page`, `.edit-container`).
2. Установите единый стиль:
   ```css
   .card, .form-container {
     background: white;
     border: 1px solid #c8c8c8;
     border-radius: 15px;
     box-shadow: 0 0 2px rgba(0,0,0,0.05);
     padding: 40px; /* Для десктопа */
   }
   .dark .card, .dark .form-container {
     background: #1e1e21;
     border-color: #2a2a2e;
   }
   ```
3. Примените те же стили к страницам авторизации `pages/login.vue` или `pages/auth.vue` (если они существуют).

### Шаг 3: Мобильная Адаптивность форм Админки
Некоторые таблицы и формы выходят за пределы экрана на смартфонах.
1. В каждом файле `.vue` внутри `pages/admin/`, найдите сетки (grid).
2. Замените жестко заданные `grid-template-columns: 1fr 1fr;` на адаптивные медиа-запросы:
   ```css
   @media (max-width: 768px) {
     .form-grid { grid-template-columns: 1fr; gap: 24px; padding: 20px; }
     .form-container { padding: 20px; }
   }
   ```
3. Оберните все таблицы (например, в списках статей) в `<div class="overflow-x-auto">`, чтобы предотвратить горизонтальный скроллинг всего сайта.

### Шаг 4: Унификация Инпутов и Label
1. Заголовки полей ввода (`label`) должны иметь одинаковый стиль:
   - Размер шрифта `12px` или `13px`, `font-weight: 700`.
   - `text-transform: uppercase`, `letter-spacing: 0.06em`.
2. Сами поля должны иметь скругления `border-radius: 10px` и легкие фокусы (box-shadow цвета `sky`). Замените стандартные инпуты, не соответствующие этому стилю, на элементы `<UInput>` или `<UTextarea>` с `size="lg"`.

### Шаг 5: Финальная проверка (QA)
После завершения шагов 1-4:
1. Запустите проект (`npm run dev`).
2. Проверьте форму добавления статьи и форму добавления книги — они должны выглядеть так, как будто собраны из одного конструктора.
3. Откройте инструменты разработчика Chrome и переключитесь на мобильное устройство (ширина 375px). Убедитесь, что нигде нет горизонтального скролла с обрывами контента.

**Ожидаемый результат:** Абсолютно целостный, премиальный интерфейс, где пользователь безошибочно понимает, какая кнопка главная, а какая второстепенная, без визуального "прыгания" стилей при переходе со страницы на страницу.
