# Конспект задачи: Мультиязычный импорт ODM (Задача 40)

Этот документ содержит анализ текущего состояния импорта ODM, наше понимание требований по задаче №40 и открытые вопросы, на которые необходимо ответить для успешной реализации.

---

## 1. Текущее состояние импорта ODM в системе

1. **Создание проекта:** 
   * Сейчас через эндпоинт `POST /api/import/odm/project` загружается **один** файл `.odm` (по умолчанию считается основной локалью, обычно English). Из него извлекается структура глав (слотов) и записывается в `odm_project_parts` в колонки `display_title` и `master_href`.
2. **Локализация структуры (Enrichment):** 
   * Существует эндпоинт `POST /api/import/odm/project/:id/enrich-locale`, который принимает отдельный `.odm` файл и параметр `locale` (`ru` / `zh`). Он сопоставляет его слоты по позиции (`sort_order`) с созданными ранее частями проекта и заполняет локализованные поля: `display_title_ru` / `display_title_zh` и `master_href_ru` / `master_href_zh`.
3. **Импорт ODT-файлов (Bulk ODT):**
   * Через `POST /api/import/odm/project/:id/bulk-odt` загружаются архивы/файлы `.odt` для конкретного языка (`lang`). Файлы сопоставляются со слотами по названию (из `master_href_{lang}`) или по позиции. 
   * При первом импорте (обычно `en`) создаются статьи с `is_published = 0`. При последующих импортах (`ru`, `zh`) обновляются локализованные поля HTML-контента (`html_content_ru`, `html_content_zh`) существующих статей.

---

## 2. Понимание требований задачи №40

Цель — сделать процесс создания мультиязычной книги через ODM быстрым и удобным за один проход.

1. **Единовременный импорт 3 ODM-файлов:**
   * Администратор должен иметь возможность загрузить сразу до 3 файлов `.odm` (для Русского, Английского и Китайского языков) при создании проекта.
   * Система должна распарсить все файлы и автоматически сопоставить их слоты по позиции (`sort_order`).
   * В базе данных для каждого слота сразу должны заполниться:
     * Названия глав на всех языках (`display_title`, `display_title_ru`, `display_title_zh`).
     * Исходные пути к файлам `.odt` (`master_href_en`, `master_href_ru`, `master_href_zh`).
2. **Упрощенная загрузка мультиязычной книги:**
   * После создания проекта админ должен иметь возможность легко загрузить пачки `.odt` файлов для всех трех языков (или по очереди для каждого языка), чтобы они автоматически распределились по соответствующим языковым версиям статей.

## 3. Открытые вопросы для обсуждения

> [!IMPORTANT]
> Для правильного проектирования интерфейса и бэкенда ответьте, пожалуйста, на следующие вопросы:

### Вопрос 1: Как должен выглядеть интерфейс загрузки ODM?
Хотите ли вы:
* **Вариант А:** Сделать три отдельных поля выбора файлов на странице создания проекта (выбрать `ODM (EN)`, `ODM (RU)`, `ODM (ZH)`)?
* **Вариант Б:** Одно поле загрузки (multiple), где файлы будут распознаваться по суффиксам в названии (например, `book_en.odm`, `book_ru.odm`, `book_zh.odm`) или по структуре внутри?
* **Вариант В:** Оставить загрузку одного главного ODM для инициализации, но добавить в интерфейсе админки удобную секцию «Локализовать структуру», где можно в один клик прикрепить оставшиеся два ODM-файла?

### Вопрос 2: Логика сопоставления при bulk-загрузке ODT
Сейчас при массовой загрузке ODT-файлов (`bulk-odt`) мы сопоставляем имя файла с `master_href_{lang}`.
* Если ODM-файлы на разных языках ссылаются на файлы с разными именами (например, `chapter1_en.odt` и `glava1_ru.odt`), гарантирует ли сопоставление по позиции (`sort_order`) 100% корректность, или нам нужно требовать строго одинаковые имена файлов ODT?
* Хотите ли вы иметь возможность загружать ODT-файлы сразу для всех языков в одном запросе (например, выделив 30 ODT-файлов разных локалей), или загрузка поочередно для каждого языка (выбрал язык -> загрузил ODT) является предпочтительной?

### Вопрос 3: Главный язык проекта (Fallback)
Если для какого-то слота загружен ODT только для одного или двух языков, как мы должны себя вести?
* Должны ли мы создавать статью с пустыми полями для остальных языков, или копировать контент из доступного языка в качестве фолбека?
